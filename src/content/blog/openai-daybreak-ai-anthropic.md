AI企業間の競争は、ついに最も機密性の高い領域である「サイバーセキュリティ」へと波及した。

5月12日、OpenAIは **Daybreak** を正式に発表した。これは GPT-5.5-Cyber と Codex Security をベースとしたサイバーセキュリティプラットフォームであり、脆弱性検知、パッチ検証、およびセキュアなソフトウェア開発に特化している。

これは単なる新製品のリリースではない。1か月前、Anthropicが「危険すぎて公開できない」と称して Claude Mythos を発表した。それに対するOpenAIの回答が今回の Daybreak であり、しかもAnthropicとは全く異なるアプローチを採っている。

---

## 1. Daybreakとは何か？

### コアコンセプト

OpenAIは公式サイトで次のように述べている：

> "Daybreakは、サイバーセキュリティにおける我々のビジョンであり、ソフトウェアの構築と防御の方法を変えることである。サイバーセキュリティにおけるDaybreak（夜明け）とは、リスクをより早く察知し、より迅速に行動することで、ソフトウェアに本来的なレジリエンスを持たせることを意味する。"

Daybreakという名前には、攻撃者が動き出す前に脅威を察知するという「夜明け」の比喩が込められている。

### 3つのコア能力

DaybreakはAIによるサイバー防御を以下の3つのフェーズに分けている：

**1. 高影響な脅威へのフォーカス**

AIにより、数時間かかっていた分析を数分に短縮する。ノイズに埋もれることなく、真に重要な脆弱性を正確に特定する。

**2. 大規模なセキュアパッチ生成**

コードリポジトリ内で直接パッチを生成し、範囲制御、監視、レビュープロセスを経てテストする。「脆弱性を見つけて報告する」のではなく、「見つけて修正する」までを完結させる。

**3. すべての修正を検証**

結果と監査可能なエビデンスをシステムに書き戻し、修正のあらゆるステップを追跡・検証する。

### 技術的基盤

Daybreakを支えるのは、2つの重要なプロダクトである：

- **Codex Security**：3月に発表されたAIセキュリティエージェント。企業のコードベースに基づいた脅威モデルの構築、攻撃パスの分析、隔離環境での脆弱性検証を可能にする。
- **GPT-5.5-Cyber**：サイバーセキュリティシナリオに特化して学習されたGPT-5.5のバリアント。安全性の制限が緩和されており、レッドチームテストやペネトレーションテストをサポートする。

## 2. 3層モデルアーキテクチャ：ユーザーに応じた権限管理

OpenAIは Daybreak に階層的なアクセス体系を導入した。ここが最も注目すべき詳細点である。

| モデル | 特徴 | 適用シナリオ |
|------|------|----------|
| **GPT-5.5（デフォルト）** | 標準的な安全ガードレール | 一般的な開発、ナレッジワーク |
| **GPT-5.5 + Trusted Access for Cyber** | より精密な安全保護、認可環境下での防御業務 | セキュアコードレビュー、脆弱性トリアージ、マルウェア分析、検知エンジニアリング、パッチ検証 |
| **GPT-5.5-Cyber** | 挙動制限を最小化、強力な検証とアカウントレベルの制御 | 認可されたレッドチームテスト、ペネトレーションテスト、制御下での検証 |

この設計は非常に戦略的だ。

**一般の開発者は GPT-5.5 で十分**であり、セキュアコードレビューや脆弱性トリアージが可能だ。

**専門のセキュリティチームは Trusted Access を申請**することで、より高度な防御能力をアンロックできる。ただし、認証情報の盗用や永続化、マルウェアの展開といった悪意ある行為を防ぐ厳格なガードレールは維持される。

**最上位の GPT-5.5-Cyber は、重要インフラのセキュリティチームのみに開放**され、強力なアカウント検証と監視の下でレッドチームテストやペネトレーションテストに使用される。

OpenAIは具体例として、デフォルトの GPT-5.5 ではCVEの脆弱性利用コード（Exploit）の生成を拒否するが、Trusted Access 環境下ではPoCの生成と修正の検証をサポートし、GPT-5.5-Cyber ではリアルタイムのターゲットテストを実行できるとしている。

## 3. Daybreak vs. Mythos：対極にある2つの戦略

OpenAIの Daybreak と Anthropicの Claude Mythos は、表面上は競合だが、実際にはAIセキュリティに対する2つの異なる哲学を体現している。

### Anthropicの路線：閉鎖、慎重、「危険すぎて公開不能」

Anthropicは4月に Claude Mythos Preview を発表し、同時に「危険すぎて一般公開できない」と主張した。その能力は驚異的だ：

- **数千のゼロデイ脆弱性を自律的に発見**し、主要なOSやブラウザを網羅した。
- 最も安全とされるOpenBSDにおいて、27年間放置されていた脆弱性を発見した。
- FFmpegにおいて、500万回の自動テストをすり抜けて16年間存在していた脆弱性を発見した。
- Linuxカーネル内の複数の脆弱性を自律的に連結し、一般ユーザーから完全なシステム制御権を得る権限昇格を実現した。

CyberGymベンチマークにおいて、Mythos Previewの脆弱性再現率は **83.1%** に達し、Claude Opusの 66.6% を大幅に上回った。

しかし、Anthropicの手法は**「厳格なアクセス制限」**である。Mythosは、AWS、Apple、Cisco、CrowdStrike、Google、Microsoft、NVIDIAなどの厳選された少数パートナーにのみ共有され、1億ドルの利用枠が割り当てられた。

皮肉なことに、これほど慎重であるにもかかわらず、4月には権限のない第三者がMythosへのアクセス権を奪取したという漏洩事件が The Verge によって報じられている。

### OpenAIの路線：開放、階層化、「拡張可能な防御エコシステム」

OpenAIは全く異なる道を歩んでいる。

Daybreakの核心は単一のモデルではなく、**拡張可能な防御エコシステム**である：

- **Cisco, Oracle, CrowdStrike, Palo Alto Networks, Cloudflare, Fortinet, Akamai, Zscaler** の8社と提携。
- **欧州委員会（European Commission）**と接触し、AIモデルを用いたサイバーセキュリティ脆弱性の特定について協議。
- 一般開発者から専門チームまで、権限に応じた階層的アクセスを採用。

EIIRTrendのアナリスト、Pareekh Jain 氏は次のように分析している：

> "DaybreakによってOpenAIは、最先端のサイバーAI領域でAnthropicがリードしているという印象を打破した。OpenAIは秘密のモデルに頼るのではなく、企業のワークフローや開発環境に組み込める拡張可能なサイバー防御エコシステムを構築している。"

### 両路線の比較

| 维度 | OpenAI Daybreak | Anthropic Mythos/Glasswing |
|------|----------------|---------------------------|
| **コアモデル** | GPT-5.5-Cyber + Codex Security | Claude Mythos Preview |
| **アクセス方法** | 階層的に開放、申請可能 | 厳格に制限、招待制のみ |
| **パートナー** | セキュリティ企業8社 + 欧州委員会 | AWS/Apple/Google/Microsoft等 + 40以上の組織 |
| **哲学** | 拡張可能な防御エコシステム | 閉鎖的なテストによる濫用防止 |
| **商用化** | エンタープライズ製品（営業経由） | 非商用、1億ドルの利用枠提供 |
| **漏洩事件** | なし | あり（4月に未認可アクセス） |

## 4. なぜ今、AIセキュリティが加速しているのか？

2026年、AIセキュリティはもはやニッチな話題ではなくなった。理由は単純だ。

**AIモデルのコード能力が、大多数の人間（セキュリティ専門家）を凌駕し始めたからである。**

Anthropicは Glasswing ブログの中で次のように述べている：

> "10年前のDARPAサイバーグランドチャレンジ以降、最先端のAIモデルは現在、脆弱性の発見と利用能力において最高レベルの人間と競い合っている。"

これは誇張ではない。Mythos Previewが数十年間発見されなかったゼロデイ脆弱性を特定し、Linuxカーネルでの完全な権限昇格を実現したことがそれを物語っている。また、Googleも同じ週に、**AIが開発した初のゼロデイ脆弱性**を特定し、大規模な利用を未然に防いだことを発表している。

つまり、**防御側と攻撃側の双方が、かつてない能力を得ている**ということだ。問題は「どちらが先に使いこなすか」にある。

OpenAIとAnthropicは共に、AIサイバーセキュリティが次なる巨大市場になると賭けている。しかし、そのアプローチは対照的だ：

- **Anthropicの賭け**：まず閉鎖的にテストし、安全を確保してから開放する。
- **OpenAIの賭け**：迅速にエコシステムを構築し、より多くの防御者にAIを普及させる。

どちらが正解かはまだ分からない。だが確かなことは、**防御者にAIツールを与えなければ、攻撃者が先にそれを手にする**ということだ。

## 5. 欧州委員会の動向に注目すべき理由

Daybreakの発表における見落としがちだが重要な点がある。**欧州委員会が、新しいAIモデルを用いてサイバーセキュリティ脆弱性を特定するため、OpenAIと協議している**点だ。

欧州委員会の広報担当者 Thomas Regnier 氏は、OpenAI側の主体的なアプローチを歓迎し、透明性と新モデルへのアクセス提供について協議が進んでいることを明かした。

これはAnthropicとの鮮明なコントラストとなっている。Anthropicは当初、米国の技術パートナーのみにMythosを共有しており、欧州との議論はOpenAIほどの段階に達していなかった。

分析者の Amit Jaju 氏は次のように指摘する：

> "OpenAIは『Trusted Access』フレームワークを戦略的に利用し、欧州の規制当局との信頼関係と透明性を迅速に構築している。早期アクセスを提供することで、今後の規制要件に適合させ、戦略的な市場ポジションを確保しようとしている。"

AI規制が厳格化するグローバル環境において、規制当局との信頼を先に築いた者が先手を取ることになる。

## 6. AIセキュリティの究極的な問い：能力と責任

DaybreakもMythosも、一つの根本的な問題に直面している。

**AIが自律的にゼロデイ脆弱性を発見し、攻撃コードを開発できるようになったとき、誰にその利用権を与えるべきか？**

OpenAIの答えは「階層的アクセス ＋ パートナー検証」であり、Anthropicの答えは「厳格な管理 ＋ 小規模共有」である。

どちらの手法にもリスクがある：
- OpenAIの開放的なエコシステムは、能力の拡散を招く可能性がある。階層制御があるとはいえ、アクセス範囲が広い分、漏洩面も広くなる。
- Anthropicの閉鎖路線は、すでに問題を露呈している。4月の未認可アクセス事件は、「鍵をかけて閉じ込めても盗まれる」ことを証明した。

より深刻な問いは、**AIが27年間未発見だった脆弱性を見つけられるようになった今、悪意ある攻撃者が同様の能力を得るまでの窓は閉じつつある**ということだ。これは「起こるかどうか」ではなく、「いつ起こるか」の問題である。

## 7. 開発者にとっての意味は何か？

開発者やセキュリティ担当者にとって、Daybreakがもたらす実用的な影響は以下の通りだ。

### 1. セキュアコードレビューのAI時代へ

GPT-5.5 + Trusted Access により、開発サイクルの中でリアルタイムなセキュアコードレビューが可能になる。コードを書き終えてからスキャンツールを回すのではなく、記述しながら脆弱性を発見できる。

### 2. パッチ検証の自動化

Daybreakの Codex Security は、コードリポジトリ内で直接パッチを生成し、テストを行う。 「脆弱性発見」から「修正検証」までのプロセスを自動化できる。

### 3. 脆弱性スキャンの申請

OpenAIはDaybreakのページで「脆弱性スキャンのリクエスト」口を開放した。企業は自社コードベースのAIスキャンを申請できる。

### 4. 適切なアクセスレベルの選択

ほとんどのセキュリティチームにとって、GPT-5.5 + Trusted Access で十分な機能が得られる。GPT-5.5-Cyber が必要になるのは、レッドチームテストやペネトレーションテストを専門に行うチームのみである。

## 8. まとめ：セキュリティ戦線の本格的な開幕

Daybreakのリリースは、AI業界の競争が新たな段階に入ったことを示している。

これまでOpenAIとAnthropicは、モデル能力、企業提携、資金調達規模で競い合ってきた。しかし、サイバーセキュリティは全く異なる戦場である。そこには国家安全保障、インフラ保護、犯罪防止が絡んでおり、「どちらのチャットボットが賢いか」という議論よりも遥かに深刻な問題だ。

OpenAIは開放的なエコシステムの道を、Anthropicは閉鎖的で慎重な道を選んだ。どちらが正しかったかは、数年後には分かるだろう。

しかし、一点だけ確かなことがある。**AIサイバーセキュリティはもはやオプションではなく、すでに始まっている競争である。防御者は攻撃者に勝ち続けなければならない。**

Daybreakという「夜明け」の名は、適切かもしれない。夜が明ける前に、まずは目覚める必要があるからだ。
